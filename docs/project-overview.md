# 프로젝트 개요

## 1. 프로젝트명
BankFlow

## 2. 프로젝트 목표
BankFlow는 **신뢰감 있는 디지털 뱅킹 경험을 시연하기 위한 금융 서비스 MVP**이다. 실제 금융 코어 시스템 전체를 구현하기보다, 사용자가 금융 서비스를 이용할 때 체감하는 핵심 흐름을 짧은 기간 안에 설득력 있게 구현하고, 동시에 AWS 아키텍처 이해도와 AI 활용 가능성을 보여주는 것을 목표로 한다.

## 3. 1주 기준 현실적인 개발 범위 재정의

### 왜 범위를 줄였는가
금융 서비스는 인증, 보안, 거래 정합성, 계좌계, 심사, 관리자 기능까지 포함하면 구현 범위가 매우 커진다. 따라서 1주 프로젝트에서는 실제 운영 수준 전체를 구현하는 대신, 아래와 같은 **핵심 사용자 경험과 설명 가능한 구조**에 집중했다.

### MVP 범위
- 메인 페이지: 서비스 가치 제안과 AWS/AI 시연 포인트 요약
- 로그인 페이지: 데모 계정 기반 로그인, 입력 검증, 세션 유지
- 보호 페이지 접근 제어: 로그인 후 대시보드, AI 챗봇, 이체 페이지 접근
- 대시보드: 자산, 소비, AI 브리핑, 최근 거래 요약
- 금융상품 페이지: 핵심 상품 3종 소개
- AI 챗봇 페이지: 상담 유형과 추천 근거를 보여주는 상담 UX 시연
- 이체 시뮬레이션 페이지: 입력, 확인, 완료 3단계 흐름 시연
- 문서화: AWS 아키텍처, AI 기능, 테스트 결과 정리

### 제외 범위
- 실제 금융기관 API 연동
- 실명 인증 및 다중 계정 권한 처리
- 실제 송금/결제 실행
- 관리자 대시보드 및 심사 백오피스
- 운영 수준 모니터링 자동화

## 4. 현재 구현 포인트

### 사용자 경험 측면
- 은행 서비스처럼 보이는 차분한 디자인 톤 적용
- 보호 페이지 접근 구조로 서비스 흐름 현실감 보강
- AI 답변에 상담 유형과 추천 근거 표시
- 이체 화면에 확인 단계와 완료 영수증 스타일 추가

### 기술 설명 측면
- Frontend, 인증, API, 데이터, AI를 분리한 AWS 구조 설명 가능
- Bedrock 연동을 가정한 상담 시나리오 제시 가능
- 로컬에서 실제 실행 가능한 데모 화면 제공

## 5. 기대 효과
- 평가 기준에 맞춘 명확한 구조 제시
- 짧은 기간 안에도 구현 가능한 서비스 설계 능력 증명
- AI 기능이 단순 아이디어가 아니라 실제 UX 흐름으로 보이도록 구성
- 로그인, 상담, 이체 흐름을 통해 금융 서비스다운 완성도 전달

## 7. 성공 기준
- 로컬에서 실제 실행 가능
- 평가 기준 4개 영역이 모두 문서와 화면에 반영됨
