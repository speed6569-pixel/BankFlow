# 프로젝트 개요

## 1. 프로젝트명
BankFlow

## 2. 프로젝트 목표
1주일 안에 완성 가능한 금융/은행 서비스 데모 웹사이트를 제작한다. 실제 금융거래 시스템 전체를 구현하는 대신, 사용자가 핵심 서비스를 체험할 수 있는 흐름과 AWS 기반 설계 역량, AI 기능 활용 능력을 보여주는 것을 목표로 한다.

## 3. 1주 기준 현실적인 개발 범위 재정의

### 왜 범위를 줄였는가
금융 서비스는 인증, 보안, 계좌계, 거래 정합성, 관리자 기능까지 포함하면 구현 범위가 매우 커진다. 따라서 1주 프로젝트에서는 아래 MVP 범위만 구현한다.

### MVP 범위
- 메인 페이지: 서비스 가치 제안과 평가 포인트 요약
- 로그인 페이지: Cognito 연동을 가정한 시연용 로그인 UI
- 대시보드: 자산, 소비, AI 추천, 최근 거래 요약
- 금융상품 페이지: 핵심 상품 3종 소개
- AI 챗봇 페이지: Bedrock 기반 상담 시나리오를 UI로 시연
- 이체 시뮬레이션 페이지: 입력 검증과 결과 확인이 가능한 거래 모듈
- 문서화: AWS 아키텍처, AI 기능, 테스트 결과 정리

### 제외 범위
- 실제 금융기관 API 연동
- 실명 인증 및 다중 계정 권한 처리
- 실제 송금/결제 실행
- 관리자 대시보드 및 심사 백오피스
- 운영 수준 모니터링 자동화

## 4. 기대 효과
- 평가 기준에 맞춘 명확한 구조 제시
- 짧은 기간 안에도 구현 가능한 서비스 설계 능력 증명
- AI 기능이 단순 아이디어가 아니라 화면 흐름으로 보이도록 구성

## 5. 주간 개발 계획

### Day 1
- 요구사항 정리
- 화면 구조 정의
- 프로젝트 초기 세팅

### Day 2
- 메인 페이지, 로그인 페이지 구현
- 공통 레이아웃 정리

### Day 3
- 대시보드, 금융상품 페이지 구현
- 더미 데이터 구성

### Day 4
- AI 챗봇 페이지 구현
- Bedrock 연동 설계 문서 정리

### Day 5
- 이체 시뮬레이션 구현
- UX 다듬기

### Day 6
- AWS 아키텍처 문서와 테스트 리포트 작성
- 실행 점검

### Day 7
- 최종 발표용 점검 및 수정

## 6. 성공 기준
- 로컬에서 실제 실행 가능
- 평가 기준 4개 영역이 모두 문서와 화면에 반영됨
- AI 기능이 UI 상에서 직접 시연 가능
- AWS 리소스 구성과 연결 관계가 명확하게 설명됨
